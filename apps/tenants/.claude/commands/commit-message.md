---
description: Descriptive commits message generation
---

Check current git working tree, analyze ONLY staged changes. Generate a short and concise commit message, summarizing all of the most valuable and important STAGED changes in 5-15 words. Ignore the chat history. Never commit yourself.

Use prefixes to categorize the changes:
- feat: new feature
- fix: bug or issue resolution
- docs: documentation updates
- refactor: code refactoring without explicit features or bug fixes
- perf: improve performance
- chore: maintenance updates like dependencies or tooling
- test: changes to test files
